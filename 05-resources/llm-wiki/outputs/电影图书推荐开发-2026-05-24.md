# 电影图书推荐—开发总结 (2026-05-24, 更新 05-25)

> 关联项目：[[03-projects/电影图书推荐/开发笔记|开发笔记]] · [[03-projects/电影图书推荐/spec|项目 Spec]] · [[03-projects/电影图书推荐/index|主页面]]

## 第一阶段：电影 Wikipedia 解析 (2026-05-24)

### 核心要点

1. **数据方案经历三次迭代**：纯 SPARQL（person 查询 429/超时）→ Wikipedia HTML 解析（每个电影节表格结构不同）→ **Wikipedia 奖项独立页面解析**（最终方案，已验证可行）

2. **最终方案核心逻辑**：每个奖项对应一个 Wikipedia 页面（如 `Palme_d%27Or`），一次 API 调用拿到全部历史数据，从表头自动识别列，从 film 单元格 `<a href>` 直取 Wikipedia 链接用于海报

3. **21 个电影奖项全部通过验证**：Cannes 6、Venice 4、Berlin 7、Oscars 4，覆盖 1990-2026，海报命中率接近 100%

4. **图书仍用 SPARQL**（8 个奖项），待改为 Wikipedia 解析

5. **缓存永不过期**，仅手动刷新或版本号变化时重取

### 关键概念

- **Wikipedia Core API** (`action=parse&prop=text`)：获取页面解析后的 HTML
- **DOMParser + querySelectorAll('table.wikitable')**：浏览器端解析 Wikipedia 表格
- **表头自动识别**：遍历 `<th>` 找 film/director/actor 列
- **rowspan 检测**：区分奥斯卡获奖行（rowspan>=3）vs 提名行
- **Oscar 获奖行背景色**：`style="background:#FAEB86"` 标记获奖者
- **单元格 `<a href>` 直链**：100% 准确获取 Wikipedia 页面标题，替代搜索 API
- **mergeFilms**：`normTitle` 按片名去重，收集 `yearList` 和 `awards[]`

## 第二阶段：图书 Wikipedia 解析 (2026-05-25)

### 核心要点

1. **8 个图书奖项全部从 SPARQL 迁移到 Wikipedia 解析**，删除全部 SPARQL 代码（`runSparql`, `buildBookQuery`, `fetchAward`, `dedup`）

2. **从自动列检测演进为配置驱动**：每个奖项在 `BOOK_PAGES` 中显式指定 `yearCol`/`authorCol`/`titleCol`，不再自动猜测列位置

3. **四个特殊奖项处理**：
   - **Nobel**：两级表头（Laureate colspan=2→Image+Name），`subAuthor:true` 定位 Name 子列；页面名 `List_of_Nobel_laureates_in_Literature`
   - **Intl Booker**：2005-2015 表无 title 列→自动按 person 处理（作者照片），2016+ 有 work 列
   - **National Book Award**：2025 年起用 `<th rowspan>` 年份，列左移一位；`winnerBg` 检测 `background:LemonChiffon` 过滤获奖者
   - **Women's Prize**：页面名 `List_of_Women's_Prize_for_Fiction_winners`

4. **UI 调整**：界面标题改为 "Itzak's selection"，Tab 改为英文（Movies/Books），暗色模式已移除

5. **调试工具**：新增 `book-test.html`（下拉选奖项逐项验证）和 `nobel-test.html`

### 关键概念

- **配置驱动列映射**：`yearCol`/`authorCol`/`titleCol` + `subAuthor`/`rowspanYear`/`winnerBg` 等标志位
- **两级表头检测**：`rows[1].querySelectorAll("th")` 获取子表头，匹配 "name" 定位实际作者列
- **rowspan 年份列偏移**：当 year 来自 rowspan th 时，authorCol/titleCol 各减 1 补偿列左移
- **国旗列过滤**：`headerImgs[]` 标记含 `<img>` 的列，titleCol 检测时跳过；data cell 含 img 且无文本时跳过
- **person 自动回退**：当 title 为空但 author 有值时，`title = author; linkFromAuthor = true`，Wikipedia 链接指向作者页
- **非数据表过滤**：`headers[yearCol].indexOf("year") < 0` 的表直接 continue

### 排错要点

- **Nobel 0 items**：页面名错误（统计表≠laureates 表）→纠正为 `List_of_Nobel_laureates_in_Literature`；两级表头 `authorCol` 指向 Image 子列→sub-header 检测+定位 Name
- **Intl Booker 国旗**：无 title 列时期 `titleCol` 回退到 country（含国旗 img）→跳过 img 列+person 自动回退，链接指作者页。image-fallback 搜索适得其反→删除
- **NBA 2025 缺失**：年份用 `<th rowspan>` 独占一行→`pendingYear` 继承；数据行列左移→`effAuthorCol = authorCol - 1`
- **NBA 2025 含提名者**：shortlist 表含 winner+finalists→`winnerBg` 检测 `background:LemonChiffon` 过滤非获奖行
- **Women's Prize 0 items**：页面名错误→`List_of_Women's_Prize_for_Fiction_winners`
- **自动检测→配置驱动**：各奖项表格结构差异太大，补丁越打越多→每奖项显式指定 `yearCol/authorCol/titleCol` + 标志位
- **Multi-line Edit 失效**：Read 展示与文件实际 whitespace 不一致→改用 Python 脚本做批量替换
