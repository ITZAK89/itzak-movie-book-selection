# 多智能体交易决策框架

## 是什么

一种用多个 AI 角色协作完成交易决策的框架。每个角色有独立的视角和分析方法，通过层层递进的辩论和裁决，产出最终交易计划。首次实践见 [[btc-multi-agent-trading-analysis-20260518|BTC 多智能体交易分析报告]]。

## 架构分层

```
第一层：分项分析（4 位分析师，独立并行）
├── 市场分析师 → 技术面（均线、MACD、RSI、布林带、VWMA）
├── 情绪分析师 → 市场情绪（新闻、社交媒体、社区讨论）
├── 新闻分析师 → 宏观与事件驱动（通胀、地缘政治、行业动态）
└── 基本面分析师 → 资产内在价值（稀缺性、流动性、开发者生态）
        ↓
第二层：对抗性辩论（牛熊双方，多轮博弈）
├── 牛市研究员 ←→ 熊市研究员（5 轮辩论）
└── 每轮必须回应对方具体论点，不可回避
        ↓
第三层：风控博弈（3 位风控师）
├── 激进派 → 追求不对称损益
├── 保守派 → 追求确认度，宁可错过
└── 中立派 → 综合优化，主动管理
        ↓
第四层：组合经理最终裁决
└── 权重评估：已确认事实 > 未来预期
```

## 核心设计原则

### 1. 视角隔离
每位分析师只拿到自己领域的数据，不被他人结论影响。避免"锚定效应"——技术分析师不知道情绪分析师的结论。

### 2. 对抗性验证
牛熊双方必须直接回应对方的具体论点，不能各说各话。暴露逻辑盲区。

### 3. 事实与预期分离
组合经理裁决的核心原则：**"交易现在是什么，而非未来可能是什么。"**
- 已确认的利空 > 潜在的利好
- 已发生的卖压 > 未来可能的回购
- 当前的技术结构 > 3-4 周后的预期信号

### 4. 执行量化
最终输出必须是可执行、可验证的交易计划：入场区间、仓位、止损、加仓条件、目标、失效条件。

## 与传统分析的区别

| 传统分析 | 多智能体框架 |
|----------|-------------|
| 单一分析师综合判断 | 多角色独立分析后对抗 |
| 结论容易受个人偏见影响 | 牛熊辩论强制暴露反面逻辑 |
| 风控常被简化为一句话 | 三方风控师独立博弈 |
| 决策依据模糊 | 权重明确：已确认 > 预期 |

## 适用场景

- 高不确定性、多空分歧大的资产
- 需要明确风险边界和止损纪律
- 团队已有不同观点的场景（可把不同观点编码进不同角色）

## 局限

- 信息密度大，决策链条长，不适合高频/短线
- 角色设定影响结论（牛熊双方的"信仰"本身就是变量）
- 框架质量取决于各角色分析师的提示词设计
