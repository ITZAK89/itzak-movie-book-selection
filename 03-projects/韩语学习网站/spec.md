# 韩语学习网站 — Spec

## 目标

一站式韩语自学工具，覆盖词汇、语法、跟读三个学习环节。纯静态 HTML，浏览器直接打开即用（跟读模块需本地 server 提供音频下载）。

## 功能清单

### 词汇 (Vocabulary)
- [x] 闪卡模式：正面韩语单词 + 等级标签，点击翻转显示中文释义、韩语释义、例句
- [x] 输入模式：显示中文释义，用户打字输入韩语，提交后判断正误
- [x] 等级筛选 (TOPIK 4/5/6) 和分类筛选
- [x] 每日学习量控制（可配置单词数）
- [x] "今日复习"和"强化复习"列表，存入 localStorage
- [x] 进度条 + 词汇总表（可展开/折叠）
- [x] 键盘导航（Enter 翻转，方向键前后切换）

### 语法 (Grammar)
- [x] 学习模式：可展开的手风琴卡片，展示语法点、释义、详解、例句
- [x] 测验模式：填空选择题，即时反馈（正确/错误 + 解释）
- [x] 等级和分类筛选
- [x] 每日学习量控制
- [x] 复习列表（今日/强化）

### 跟读 (Shadowing)
- [x] 粘贴 YouTube URL + VTT 字幕内容
- [x] 后端 server.py 调 yt-dlp 下载音频；也支持直接上传音频文件
- [x] 音频播放器，字幕同步显示
- [x] 每条字幕：显示/隐藏切换，点击跳转音频到对应时间戳
- [x] "已学会"标记（按视频 ID + 内容哈希去重）
- [x] 完整字幕列表，可点击导航
- [x] 键盘快捷键（Enter 显示，方向键导航）
- [x] 学习状态全部存入 localStorage

### 全局
- [x] 中英双语 UI，顶部栏切换
- [x] 学习连续天数统计（localStorage），悬停显示各模块学习量
- [x] "清除全部"按钮重置表单

## 数据结构

### 词汇 (381 条)
```
{
  word: "가능성",        // 韩语
  meaning: "可能性",      // 中文释义
  eng: "possibility",    // 英文释义
  level: 4,              // TOPIK 等级 4/5/6
  category: "抽象概念",   // 分类
  kr_defs: ["...", "..."],  // 韩语词典释义
  examples: [{ ko: "...", zh: "..." }]  // 例句
}
```
数据源：`batch1.json` ~ `batch7.json`（按分类分批）

### 语法 (27 条学习 + 27 条测验)
```json
// 学习
{ "pattern": "-기 때문이다", "meaning": "因为...", "explanation": "...", "examples": [...], "level": 4, "category": "..." }
// 测验
{ "question": "选择正确的语法:", "options": [...], "answer": 0, "explanation": "..." }
```
数据源：`grammar_study4.json` / `grammar_quiz4.json`

### 跟读 (用户自供内容)
数据不预置，用户粘贴 YouTube URL + VTT 字幕，或上传音频文件。学习状态存 localStorage。

## 技术选择

| 决策 | 选择 | 原因 |
|------|------|------|
| 框架 | 无，纯 vanilla HTML/CSS/JS | 零依赖，file:// 即用 |
| CSS | 内联 `<style>`，约 160 行 | 单文件自包含 |
| 数据 | 嵌入为 JS 数组 | 无网络请求，即时加载 |
| 持久化 | localStorage | 无需后端数据库 |
| 音频服务 | Python `server.py` (8766 端口) + yt-dlp | 仅跟读模块需要 |
| 构建 | `regenerate_html.py` 从 JSON 重新生成 HTML | 避免手工编辑 1500 行 HTML |
