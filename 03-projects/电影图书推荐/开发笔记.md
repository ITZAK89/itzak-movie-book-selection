# 电影图书推荐 — 开发笔记

## 项目概述

聚合 Wikipedia / Wikidata 实时数据的电影和图书推荐网站。纯静态 HTML，file:// 直接打开。

## 访问方式

- 本地：`open "03-projects/电影图书推荐/index.html"`
- 本地 server：`python3 -m http.server 8766` → `http://localhost:8766/03-projects/电影图书推荐/index.html`
- 测试页：`test.html`（API 连通性验证）
- 诊断：浏览器 Console 运行 `runDiag()`（已移除，待恢复）
- 离线模式：`test-data.js`（导出后用 Export 按钮，gitignore）

## 架构决策

### 数据源

| 类型 | 来源 | 方式 |
|------|------|------|
| 电影 work 奖（7个） | Wikidata SPARQL | `?item wdt:P166 wd:Qxxx` + `wdt:P31 wd:Q11424` |
| 电影 person 奖（11个） | Wikidata SPARQL | `?person wdt:P166 wd:Qxxx` + `pq:P1686 ?work` |
| 图书（8个） | Wikidata SPARQL | work/person/person_simple 混合 |
| 海报/封面 | Wikipedia REST API | `/page/summary/{title}` → `thumbnail.source` |
| TSPDT | JSON 快照 | 未实现 |
| 媒体书单 | JSON 快照 | LibraryThing CORS 不通，未实现 |

### 去重策略

- `mergeFilms()`: 按 `normTitle(title)` 去重，跨年份合并，收集 `yearList` 和 `awards[]`
- work 查询加 `wdt:P31 wd:Q11424` 过滤，排除人物实体
- person 查询跳过无 `workLabel` 的行

### 缓存策略

| 层级 | TTL | Key |
|------|-----|-----|
| 单奖项 SPARQL | 24h | `award_{qid}` |
| 完整数据集 | 1h | `full_movies` / `full_books` |
| Wikipedia 摘要 | 7d | `wiki_enrich` |
| 版本号 | 永久 | `mbr_version`（变更时清除所有缓存） |

### 筛选 UI

- 两级：来源行 + 年份行
- 默认显示最新 10 部（All + All 年份）
- 选择具体来源或年份后不限数量

## 开发历程

### 阶段 1: API 验证
- test.html 验证 Wikipedia REST / Core / Wikidata SPARQL 连通性
- 确认正确的 Q-ID（多个 Q-ID 经过修正）
- 发现 person 查询 pq:P1686 超时问题（部分 >60s）
- 中译本 SPARQL 超时（65s），决定默认假设有中译本

### 阶段 2: 初版 HTML
- 单文件 HTML，内联 CSS + JS
- 数据加载、卡片渲染、弹窗、收藏
- 暗色/亮色主题

### 阶段 3: 缓存 & 去重
- localStorage 多层缓存 + 版本号清理
- 跨年份影片去重合并
- 两级筛选（来源→年份）

### 阶段 4: Q-ID 验证

- 逐个打开 Wikidata 页面，人工确认全部 18 个 Q-ID
- 修正：Grand Prix Q28455848→Q844804, Jury Prize Q1342432→Q164200
- 确认：Cannes Director Q510175, Actor Q586140, Actress Q840286 均正确
- 制作 qid-verify.html 方便后续验证

### 阶段 5: 数据方案反复

**问题**: person 模式 SPARQL 用错 Q-ID → 全返回空

**错误转向**: 改 Wikipedia HTML 解析
- 每个电影节页面结构不同
- Cannes 表头 `award | winner | film`，Oscars `awards | film`
- 429 限流 + 页面找不到（Venice/Berlin 用年份命名而非序数）
- 复杂度爆炸，弃用

**最终方案**: 回到纯 SPARQL
- 修正所有 Q-ID（Cannes Director Q510175, Actor Q586140, Actress Q840286）
- 查询间加 2s 间隔防 429
- person 查询 60s 超时，work 查询 30s
